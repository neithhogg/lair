# lair — Project State

## Current State

Foundation layer for AI-assisted projects. Provides a set of skills and rules that install into Claude Code to give any project a consistent working method. The repo is self-referential: lair is built using its own skills and follows its own rules.

**Status:** Foundation complete. Skills and rules written. Ready for first real-world use.

## What This Is

- A set of Claude Code skills installable via `npx skills add neithhogg/lair`
- Four skills: `project-init`, `project-sync`, `skill-writer`, and the root `lair` skill
- Four rule files that Claude Code injects as context after install
- Self-referential: this repo has its own `PROJECT.md` and `JOURNAL.md`, generated by the same patterns the skills teach

## What This Is Not

- Not a framework or library
- Not a CLI tool
- Not opinionated about tech stack (the skills help you choose, not choose for you)
- Not a replacement for documentation — it's a scaffold for building it

## Tech Stack

- Pure Markdown — no code, no dependencies, no build step
- Skills are SKILL.md files following the skills.sh specification
- Published via GitHub; installed via `npx skills add`

## Scope Boundaries

**In scope:**
- Project initialization ritual (project-init)
- Project state tracking (project-sync)
- Skill authoring assistance (skill-writer)
- Rules for file management, git, coding conventions, AI behavior

**Out of scope:**
- Language-specific scaffolding (handled by skills installed during project-init)
- CI/CD configuration
- Deployment or infrastructure
- Testing frameworks

## Constraints

- Every SKILL.md must stay under 500 lines (skills.sh requirement)
- No hardcoded skill recommendations — always search live with `npx skills find`
- Rules are static and auto-injected; they don't need to be invoked manually
- `JOURNAL.md` is append-only — never overwrite past entries
